## How to Implement

### Option 1: Zapier (No Code)
1. Create Zap: "Form Submission" â†’ "Send Email (Gmail)"
2. Set up trigger: Landing page form â†’ Zapier webhook
3. Create automation rules for delays (Day 2, Day 5, etc.)
4. Test with real submissions

### Option 2: Make.com
Similar to Zapier, with more flexibility for complex workflows.

### Option 3: Custom Backend (API)
Use SendGrid API + Node.js to send emails based on timing and user actions.

### Tracking
Add UTM parameters to all links:
- `?utm_source=email&utm_campaign=trial_day2`
- `?utm_source=email&utm_campaign=demo_followup`

This lets you see which emails drive signups and conversions.

---

## SMS Notes

- Use Twilio API or Zapier for SMS sending
- Keep messages short (<160 characters = 1 SMS)
- Always include opt-out ("Reply STOP to unsubscribe")
- Best times: 9-11am, 2-4pm (when people check phones)
- Avoid nights/weekends unless urgent

---

## Email Deliverability Tips

- Use branded domain (notifications@revenuelock.com, not Gmail)
- Add SPF/DKIM records
- Include unsubscribe link (required by law)
- Don't send from no-reply address (use real person name)
- Test subject lines on {free tools}
- Warm up domain before high-volume sends

---

## Metrics to Track

- **Open rate** (target: >25%)
- **Click rate** (target: >5%)
- **Reply rate** (target: >2% for cold outreach)
- **Conversion rate** (form submission or calendar book)
- **Unsubscribe rate** (should be <0.5%)

Adjust sequences based on data. If a subject line gets 40% opens, use that template for others.
